# GPT 反代教程：使用 Cockpit 导入账号并启动 API 服务

通过账号密码登录通常还需要接收验证码，操作比较麻烦。卡密兑换完成后，网站会提供一个 JSON 文件，将它转换为 Cockpit 支持的格式后即可直接导入使用。

本文主要介绍以下内容：

- 将 CPA/Sub2 格式的 JSON 文件转换为 Cockpit 格式
- 下载并安装 Cockpit
- 批量导入账号并启动 Codex
- 配置 API 服务，实现账号自动切换
- 恢复不可见的聊天记录

## 一、转换 JSON 文件格式

不同软件使用的 JSON 格式并不通用。卡网激活网站提供的是 CPA 和 Sub2 格式，因此在导入 Cockpit 之前，需要先完成格式转换。

1. 打开 [GPTSession2CPAandSub2API](https://gtxx3600.github.io/GPTSession2CPAandSub2API/)。
2. 在页面中选择 `Cockpit`。
3. 点击“选择文件”，导入从卡网下载的 JSON 文件。该工具支持批量选择。
4. 在页面上方选择目标格式 `Cockpit`。
5. 转换完成后，点击“下载 JSON”。

![JSON 格式转换页面](https://qn.ldxp.cn/43/6fabedc937784ea96490312d1813e7.png)

下载转换后的文件，稍后将它批量导入 Cockpit。

## 二、下载并安装 Cockpit

打开 [Cockpit Tools Releases](https://github.com/jlcodes99/cockpit-tools/releases)，根据自己的操作系统和处理器架构下载安装包。

本文使用的是 Windows 64 位版本 `v0.23.9`，对应的安装包为：

```text
Cockpit.Tools_0.23.9_x64-setup.exe
```

旧版本的界面和本文截图一致。如果新版功能或界面已经发生变化，可以根据实际页面调整操作；不确定该下载哪个文件时，先确认自己的系统类型和处理器架构。

安装完成后，打开 Cockpit：

![Cockpit 主界面](https://qn.ldxp.cn/13/627206669778541521a9e711e217fb.png)

## 三、导入账号并启动 Codex

1. 点击左侧第四个 `Codex` 按钮。
2. 点击页面中的蓝色加号。
3. 选择“导入”。
4. 选中转换后的 JSON 文件并完成导入。这里同样支持批量选择。

![在 Cockpit 中添加账号](https://qn.ldxp.cn/f0/aba3a87d54474e70044de9365b88fc.png)

![批量导入 JSON 文件](https://qn.ldxp.cn/22/aa8d96eeca276182aa642b5fdeb03a.png)

导入完成后，账号会出现在列表中。第一次启动前，先打开设置，找到 Codex 启动路径并完成选择。如果不清楚安装位置，可以尝试使用软件提供的默认自动选择功能。

配置完成后，点击对应账号的“启动”按钮即可。

![导入后的账号列表](https://qn.ldxp.cn/0e/7522c970da559d44e390f135a8678f.png)

::: tip 启动后仍提示登录
如果启动单个账号后，Codex 仍然要求登录，通常是 Cockpit 的兼容问题。可以将账号加入 API 服务，再通过 API 模式启动。
:::

## 四、配置 API 服务

单账号模式下，当前账号额度用完后，需要关闭 Codex，再回到 Cockpit 手动切换账号。API 服务可以统一管理多个账号，并在一个账号额度耗尽后自动切换到其他可用账号。

配置方法如下：

1. 进入 API 服务页面。
2. 点击“添加账号”。
3. 关闭对 Free 账号的使用限制。
4. 点击“全选”，选中需要加入服务的账号。
5. 点击“保存”，返回 API 服务页面。
6. 点击“启动 API”。

![API 服务页面](https://qn.ldxp.cn/3f/21d544eb37211d62a36a02b6b2f1b2.png)

![选择并添加账号](https://qn.ldxp.cn/53/ca2a7025d81af39f75c0d60ac0ccec.png)

API 服务页面提供“标准”和“快速”两种模式，可以按需切换。即使导入的都是 Free 账号，也可以通过“快速”模式使用 1.5 倍速。

## 五、切换模式时的注意事项

不建议频繁切换 API 服务和单账号模式。确定使用其中一种模式后，尽量保持不变，否则部分聊天记录可能会暂时不可见。

如果聊天记录消失，可以按下面的步骤恢复：

1. 打开“会话管理”。
2. 选中需要恢复的聊天线程，建议直接全选。
3. 点击“恢复可见性”。

![恢复聊天记录的可见性](https://qn.ldxp.cn/c5/e33aae46b6fb1fbc6131197fc60e90.png)

每次恢复可见性时，Cockpit 都可能将聊天记录重新下载到以下目录：

```text
C:\Users\<你的用户名>\.codex
```

反复恢复可能产生重复文件。清理前请先备份整个 `.codex` 目录，避免误删原始聊天记录。不了解目录结构时，保留这些文件比手动删除更稳妥。
